# Security policy

## Supported releases

Security fixes are provided for the newest LinGet release. Pre-release builds are for evaluation and may receive a replacement build instead of a patch release.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability. Use GitHub's private vulnerability reporting for this repository: **Security → Report a vulnerability**.

Include the affected LinGet version, Linux distribution, provider, reproduction steps, impact, and any suggested mitigation. Do not include access tokens, private package names, or complete system inventories. We aim to acknowledge a report within 5 business days and will coordinate disclosure after a fix is available.

## Release verification

Release archives include SHA-256 checksums, SPDX and CycloneDX SBOMs, and GitHub build-provenance attestations. See [Release artifacts](docs/release-artifacts.md) for verification commands.
