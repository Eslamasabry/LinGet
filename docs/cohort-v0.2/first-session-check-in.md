# Anonymous first-session check-in

Use this as a structured private form. No field accepts free text. Never paste commands, packages, output, receipts, names, contact details, usernames, hostnames, or paths. Delete the response after verified aggregation.

| Field | Allowed response |
| --- | --- |
| Session code | Random six-character code generated by the local form after consent |
| Consent confirmed before action | `yes` / `no — discard response` |
| Broad environment | `Debian or Ubuntu family` / `Fedora or RHEL family` / `Arch family` / `other or prefer not to say` |
| Primary provider reviewed | `APT` / `Flatpak` / `npm` / `multiple` / `none` |
| How the session was found | `project release or public post` / `shared by another user` / `other or prefer not to say` |
| Reached Today after detection | `yes` / `no` |
| Plan-comprehension rubric | `pass` / `not yet` |
| Operation outcome | `verified receipt` / `safe cancellation or read-only` / `failed before receipt` |
| Mutation before explicit confirmation | `no` / `yes` / `unsure` |
| Unexplained package change | `no` / `yes` / `unsure` |
| Chosen task result | `completed without navigation help` / `completed with navigation help` / `not completed` |
| Installation-environment help | `none` / `needed` |
| First-session usefulness | `1` / `2` / `3` / `4` / `5` / `prefer not to say` |
| Continue to day seven | `yes` / `no — withdraw and discard` |

The facilitator applies the rubric; the participant’s spoken answer is not transcribed. Aggregate `yes` and `unsure` safety answers as reports. “Shared by another user” contributes only to the aggregate referred-first-session count and is never linked to the sharer.
