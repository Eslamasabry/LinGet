# Ten-user cohort outreach copy

Use existing Linux communities or direct channels; do not purchase contact lists or export account details. Keep recruitment separate from evaluation data.

## Public community post

> **Help evaluate LinGet’s terminal-first package workflow**
>
> We are looking for ten Linux users to try the signed `v0.2.0-rc.1` prerelease on a non-critical machine or disposable VM. The session takes about 20 minutes, with a 3-minute structured check-in seven days later. You choose a package task you already intended to do and may stop or cancel at any time.
>
> Evaluation records contain only anonymous multiple-choice outcomes. We do not collect package lists, commands, terminal output, machine/account identifiers, contact details, or free-text feedback. Participation and later sharing are optional and uncompensated.
>
> Read the risk, consent, and installation guide before deciding: https://github.com/Eslamasabry/LinGet/blob/master/docs/cohort-v0.2/participant-guide.md
>
> To volunteer, use the existing reply or direct-message mechanism with only “interested”. Do not send system details.

## One-to-one invitation

> I’m helping run a ten-person evaluation of LinGet’s terminal interface. It is experimental package-management software, so the test must be on a non-critical Linux machine or VM. It takes about 20 minutes plus a short day-seven check-in, and you can stop at any time. The study keeps only anonymous structured outcomes and does not collect commands, packages, machine details, contact details, or free-text feedback. If you want to consider it, read this first: https://github.com/Eslamasabry/LinGet/blob/master/docs/cohort-v0.2/participant-guide.md. Reply only “interested” or “no”; please do not send system information.

## Screening response

Send the participant guide to interested people. Ask only these structured questions:

1. Are you 18 or older? `yes / no / prefer not to say`
2. Do you have a non-critical Linux system or disposable VM available? `yes / no`
3. Can you use a terminal without screen sharing? `yes / no`
4. Can you complete both the first session and day-seven check-in? `yes / no`

Recruit only an unambiguous `yes` to all four. Store no screening response after the cohort slot is assigned or declined. Do not ask which packages are installed, who employs the person, where they live, or why they answered.
